Установка:

1) Кинуть папку с мониторингом в корень сайта (site.ru/monAJAX/)

2) Определиться с методом размещения этого мониторинга
	- подгрузка шаблона:
		Для уставноки всего лишь требуется настроить конфиг, в главном случае указав путь для сохранения шаблона 
		(указать относительный путь до папки с Вашим шаблоном) ('template' => '...), а также 'tphp' => true, и вызвать единоразово updmon.php для формирования шаблона
	- cron (обновление данных в фоновом режиме):
		Наиболее оптимальный вариант. Сделать вариант "подгрузка шаблона", за исключением 'tphp' => false.
		Далее необходимо в панели управления хостингом сайта найти "планировщик задач" либо просто "cron". В нем, следуя по инструкциям из справки добавить файл ajax.php (ежеминутно)
	- iframe (http://htmlbook.ru/html/iframe):
		Метод примитивный. Тем не менее, им до сих пор пользуются. Для установки поребуется в указанном месте на сайте вставить HTML код:
		<iframe src="http://site.ru/monAJAX/updmon.php" width="468" height="200" frameborder="no" scrolling="no">Здесь должен был быть мониторинг ;(</iframe>
		И дописать в начало файла monStyle.css  строку:
		body{margin:0 !imprtant}



При возникновении ошибок дать файлам в папке право 777 (далее список):
     P.S. Обратите внимание, что не обязательно выдавать сразу все права по списку, можно идти по нему, последовательно выдавая право 777, если что-то в мониторинге продолжает не работать.
	- ajax.php
	- config.php, updmon.php, updmon.min.php
	- cache/, cache/cahce.json, cache/common.json, work.temp (его как бы нет, но если все еще ничего не работает, то создайте его и дайте права)
	- template/, template/common.tpl, template/server.tpl
